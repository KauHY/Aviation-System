基于Python+区块链的民航客机检修记录存证系统

采用公私钥身份认证机制，确保检修记录安全、透明、不可篡改

系统核心定位

沿用"区块链不可篡改性+民航检修存证"核心逻辑，以Python为主开发语言，搭配Solidity智能合约，适配"总负责人-管理人员-技术人员"三级角色，集成公私钥身份认证机制（免二次登录），聚焦"技术人员创建-管理人员审批-总负责人管控"核心流程，确保数据安全、权责清晰、操作高效。

核心功能模块

🔐

公私钥身份认证

用户注册时自动生成专属公私钥对，私钥由用户自主保管，公钥作为唯一身份标识上链，与用户角色永久绑定。所有操作均需通过私钥签名验证，确保操作不可伪造。

📋

维修记录管理

技术人员录入维修记录，自动生成唯一Hash ID，上链存储。支持记录创建、审批、编辑/删除，全程留痕可追溯。状态流转：Pending（待审批）→ Released（已放行）。

✅

分级审批流程

基于公私钥签名的审批机制，管理人员执行互检签名、必检签名、放行签名，审批信息上链留存。支持审批驳回，驳回原因通过私钥签名上链。

🔍

多维度查询追溯

支持按工作单号、飞机注册号、技术人员公钥/工号查询。前端展示记录完整流转链路，支持导出含公私钥签名验证信息的PDF凭证。

👥

角色权限管理

适配三级角色：总负责人（系统管控）、管理人员（审批审核）、技术人员（记录创建）。基于公私钥身份绑定，确保权限清晰，操作安全。

🛡️

安全保障机制

私钥仅用于签名验证，不存储在系统服务器；公钥上链公开，用于身份识别与操作追溯；所有签名信息同步上链，全程可追溯。

角色与权限设计

角色 核心定位 具体权限范围
总负责人 系统终极管控者 1. 合约部署与初始化，配置初始用户公私钥与角色映射；
2. 记录管控：通过私钥签名编辑未放行记录、删除违规记录；
3. 权限管理：添加/移除管理人员、技术人员；
4. 全量审计：查询所有维修记录、审批日志、操作历史；
5. 紧急管控：冻结异常记录流转，处理审批争议。
管理人员 流程审批与数据审核者 1. 审批操作：通过私钥签名执行互检、必检、放行签名；
2. 数据审核：校验技术人员创建的记录完整性，驳回无效记录；
3. 权限查询：查看管辖范围内所有维修记录，追踪流转状态；
4. 反馈提交：通过私钥签名上链提交系统优化建议。
技术人员 记录创建与一线执行者 1. 记录创建：录入飞机基础信息、动态维修数据，勾选"是否必检项目"；
2. 签名操作：完成维修后通过私钥执行"工作者签名"；
3. 数据补充：未放行记录可通过私钥签名补充动态数据；
4. 权限查询：查看本人创建/参与的记录，查询所负责飞机的历史维修数据。

系统架构设计

1

区块链层

基于Ethereum（Hardhat）私有链环境，核心为Solidity智能合约。存储维修记录、签名信息（公钥、时间戳）、用户公私钥与角色映射表、操作日志。执行权限校验与业务逻辑，提供零Gas费机制，降低使用成本。

2

后端服务层（Python主导）

技术栈：Python 3.x + Flask/Django、web3.py、cryptography。将智能合约函数封装为RESTful API，校验前端录入数据，格式化时间戳，生成Hash ID，处理公私钥签名验证，维护用户登录状态，关联公钥与角色权限。

3

前端应用层（Python主导）

技术栈：Python + PyQt5/Tkinter（桌面端）或Django Templates/Flask + Vue.js（Web端）。提供公私钥备份/找回、用户登录、签名验证工具，以及各角色专属功能模块：技术人员记录录入、管理人员审批操作、总负责人系统管控。
